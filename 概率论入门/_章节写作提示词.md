# 全书第一性原理(贯穿始终,反复回扣)

> **概率论里每一个概念,都有两副面孔:一副是冰冷的公式(读者会算的那副),一副是看得见的直觉(读者该懂的那副)。全书从头到尾只做一件事——在两副面孔之间搭桥,让你看清"这个公式 = 那个直觉"。**

读者"会算不懂",正是因为教材只给了公式那副面孔。你的工作,是把每个概念**先还原成直觉**(它在描述什么现实场景、回答什么问题),再让公式成为理解的**副产品**。

而所有概念,共享一个根本惊奇,也是全书主线:**单次随机不可预测,但大量随机服从铁一般的规律。** 每写一章,先问自己:这一章的概念,是在**驯服随机性的哪一步**?把答案写进"为什么"里。

# 核心说明:不硬套比喻,用真实场景讲直觉(与线代的关键区别)

线代用一个"橡皮膜"贯穿全书,是因为线代概念极度反直觉。**概率论不需要。** 它的概念天然长在生活里:扔骰子、抽签、赌博、质检、抽样、推荐算法……

所以本书**不强求一个贯穿比喻**。讲每个概念时,**挑最贴切的真实场景**把"为什么"讲透:

- 讲**古典概率** → 扔骰子、摸球、抽签。
- 讲**几何概率** → 约会碰面(两人随机到,等多久能遇上)、投针。
- 讲**条件概率/贝叶斯** → 体检假阳性、垃圾邮件过滤、医疗诊断。
- 讲**二项/泊松** → 扔 n 次硬币数正面、呼叫中心电话数、代码 bug 数。
- 讲**指数分布** → 等公交车、无记忆性(等了 10 分钟,还要等多久的分布不变)。
- 讲**正态/中心极限** → 身高、测量误差、把一堆随机数加起来。
- 讲**期望** → 长期赌博的平均收益、保险定价。
- 讲**协方差/相关** → 身高与体重、冰激凌销量与溺水(伪相关)。
- 讲 **MLE / 贝叶斯 / 熵 / 逻辑回归** → 训练分类器、垃圾邮件识别、推荐系统。

> **一个松散的锚(全书反复回扣,但不强套所有概念):"扔很多次,频率趋近概率。"** 每当一个概念显得抽象,回到"扔很多次会怎样",往往就通了。这是大数定律的字面意思,也是全书最根本的直觉。

# 核心写作原则(最重要,反复对照)

1. **"为什么"永远先于"怎么做"。** 全书最高原则。每个概念都要先回答:
   - **它在直觉上描述什么?**(先用真实场景讲清它在回答什么问题)
   - **不这样理解会怎样?**(只会算的人会卡在哪、踩什么坑)
   - **所以这样看。**(公式是这个直觉的速记,是理解的副产品)
   - 然后才落到公式/算法。
2. **直觉优先,公式是副产品。** 任何公式(期望、贝叶斯、方差、p 值、熵)都必须**先有直觉,再有公式**,最好能让读者自己"推出"公式。严禁上来就甩公式。
3. **大白话 + 真实场景例子。** 见上节,挑最贴切的场景。
4. **小白友好,先卸包袱。** 不假设读者记得教材定义。任何术语首次出现一句话点破。开篇类章节要明说"不懂不是你的错,是讲法反了"——这是对读者最重要的承诺。
5. **每章配真实分布图 + 模拟佐证。** 见下方"配图工作流"和"模拟佐证"。
6. **回扣全局。** 每章结尾回到"**随机 vs 规律**"主线,说明本章概念在"驯服随机性"的哪一步,衔接前后章节。

# 深度方针:越深越好,但"由浅入深、深入浅出"

读者明确要求:**内容要尽量深(含推导、进阶、测度论彩蛋、贝叶斯/ML 巅峰内容),但路径必须由浅入深、最终能浅出。** 写每一章都要自检三件事:

- **浅入口**:从最直观的真实场景起步(扔骰子 / 一次赌博 / 一份体检报告 / 一行代码),别一上来就硬核。
- **深落点**:在直觉之上,给出真正有分量的深度——公式为什么这么定义、反例、连续/测度推广、与前后概念的暗线勾连、机器学习里的真实落点。**不要停在科普。**
- **浅出收束**:章末用一句话或一个场景把全章浓缩成可记忆的 nugget,让深的也能被带走。

**彩蛋机制(兑现"越深越好")**:在合适的章节,加一节进阶——比如把概率推广到测度论(P1-02)、用熵统一信息论(P6-18)、揭示交叉熵 = MLE 的对偶(P6-18)、贝叶斯与频率派哲学之争(P5-17)。让读者看到概率论是"数学/统计/机器学习"三界的通用语言。

# 每章的标准结构(严格照此组织,顺序不可变)

1. **章首**:
   - **核心问题**(一句话)+ "读完本章你会明白……"(3~4 条,每条戳一个"为什么")。
   - 一段简短引子,把上一章的线索接过来。
   - 难度高、信息密度大的章节(贝叶斯、中心极限、熵、逻辑回归),额外加**逃生阀**:`> **如果一读觉得太难**:先只记住三件事——①…;②…;③…。`
2. **章首·一句话点破**:给一个核心结论/金句,然后说"这句话是结论,不是理由;本章倒过来拆"。
3. **正文·若干小节**:每节遵循 **"提出问题 → 不这样会怎样 → 所以这样看 → (模拟佐证/配图)"** 四段式,可用小标题分割。**招牌句式**反复用:
   - `> **直觉**:……`(用真实场景讲)
   - `> **不这样(看/理解)会怎样**:……`
   - `> **所以这样看**:……`
   - `> **钉死这件事**:……`(关键结论用这个收口)
4. **模拟佐证**(正文之后、小结之前):给 **2~3 个纸笔可算的小例子 + 一段 Python 蒙特卡洛模拟**,亲手验证"直觉 = 公式"。**这是概率论书的招牌,每章必有**——模拟要可跑(`numpy.random` / `scipy.stats`),扔它十万次,看频率趋近概率、模拟均值趋近期望、直方图趋近理论曲线。
5. **章末小结**:
   - 用真实场景/主线回顾本章;
   - 回扣全书主线(随机 vs 规律;本章在驯服随机性的哪一步);
   - **五个"为什么"清单**(读者若只记五件事);
   - "想继续深入该往哪钻"(可视化资源 + Python 实验 + 进阶);
   - 一句话衔接、引出下一章。

# 配图工作流(概率论特色 —— 图就是它的"可验证物")

概率论是高度可视化的学科,**每章必须配真实分布图/模拟图**(分布曲线、PMF 柱状、模拟直方图、频率收敛曲线、联合分布热力图等),**严禁只用 ASCII 图**。流程如下:

1. **共享工具**:所有绘图都 `import` 自 `概率论入门/figures/_plot_utils.py`(提供配色、`save`、以及 `plot_pmf`/`plot_pdf`/`sim_hist`/`convergence`/`heatmap2d` 等概率论常用 helper)。**先读这个文件,了解可用工具。**
2. **独立脚本,绝不碰共享文件**:为你的章节写一个**独立**脚本 `figures/fig-{章}-gen.py`(例:`fig-13-gen.py`,全局章号)。脚本顶部:
   ```python
   import sys, os
   HERE = os.path.dirname(os.path.abspath(__file__))
   sys.path.insert(0, HERE)
   from _plot_utils import (save, GREEN, RED, BLUE, PURPLE, ORANGE, GRAY,
                            plot_pmf, plot_pdf, sim_hist, convergence, heatmap2d)
   import numpy as np
   import matplotlib
   matplotlib.use("Agg")
   import matplotlib.pyplot as plt
   ```
   **严禁修改 `_plot_utils.py`(共享,并行安全靠它只读)。**
3. **图内标注一律用英文**(避免中文字体乱码),正文用中文。
4. **运行生成**:`python figures/fig-{章}-gen.py`,产出 `figures/fig-{章}-{序}-{主题}.png`。
5. **嵌入章节**:用 `<img>` 标签控宽(markdown 原生 `![]` 控不了尺寸,会撑满):并列/组图 `width="560"`,单图 `width="420"`。例:
   ```
   <img src="figures/fig-13-1-convergence.png" width="560" alt="图 13.1 · 大数定律:扔 n 次,正面频率趋近 0.5">
   ```
6. **图必须画对**:所有概率/分布值,先用 `scipy.stats` 或解析公式算对再画;模拟用 `numpy.random` 固定随机种子(可复现),模拟直方图要和理论曲线吻合。**画错分布/模拟写错,是概率论配图最致命的硬伤。**

> 配图建议数量:基础章 1~2 张;分布/极限/多变量密集的章(常见分布、中心极限、联合分布)2~3 张。

# 写前必做:数值与模拟核查(防止算错 / 模拟写错,这一步不能省)

数学书没有源码,但有同样致命的硬伤——**公式算错、概率值写错、模拟代码写错、图与正文对不上**。引用 / 举例前必须核实:

1. **每个概率值、期望、方差、分位数,先用 `scipy.stats` 或手算核对**(尤其正态分位数、泊松概率、贝叶斯后验),再写进正文。**严禁凭印象写数字。**
2. **每段模拟代码,先在本地跑通**,确认输出(频率/均值/直方图)与理论吻合,再写进正文。模拟务必**固定随机种子**(`np.random.default_rng(42)`),保证可复现。
3. **每张配图,必须和正文数字一致**:理论曲线、模拟直方图、收敛终点,要经得起核对。
4. **不确定的结论**(尤其涉及"对所有分布成立"的断言),宁可不写或标注"直观理解,严格证明见…",也不要拍脑袋下结论。

## 并行分组建议(避免依赖打架)

依赖关系:各篇的第 1 章主要依赖"第 0~1 篇(概率/条件概率/贝叶斯/随机变量/期望方差)";篇内章节有顺序。

**推荐分批:**

1. **先写 P0-01(概率到底在量什么)** —— 定调章,确认风格。**这一章建议单独先写**,别并行。
2. **定调成功后,可并行开多个 agent**,每篇按内部顺序推进。互不冲突的最省心分法:**一个 agent 负责一篇**,篇内按章序写(P1→P1→…;P2→P2→…)。
   - 可同时启动的"各篇第一章":P1-02、P2-05、P3-08、P4-11、P5-15、P6-18(都只依赖概率/随机变量地基)。
3. **跨篇可并行**(不同篇之间相对独立),但**篇内按顺序**(如中心极限 P4-14 依赖大数定律 P4-13 的直觉;MLE P5-15 依赖分布与似然;逻辑回归 P6-19 依赖伯努利与 MLE、交叉熵 P6-18)。

> 每个 agent 拿到任务后,**先读总纲 + 自己篇相邻章 + 一篇样本**,再动笔。
