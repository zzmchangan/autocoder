# 《Linux 同步原语深入浅出:RCU 凭什么这么快》—— 目录与导读

> 一本写给"写过 Linux 多线程程序、甚至翻过 `kernel/locking/`、`kernel/rcu/`,却总觉得一知半解"的人的小书。
>
> **一句话主旨**:多核共享可变数据,要既不出错(一致性/可见性/有序性)又不被锁拖死(竞争/上下文切换/缓存行乒乓)。阻塞睡眠锁(mutex/rwsem/futex)等不到就睡,自旋/无锁(spinlock/RCU/per-cpu/seqlock)要么死等要么根本不锁——内核用几十年工程把它们磨到极致,每一步都既快又 sound。
>
> **二分法**(迷路时回到它):**阻塞睡眠锁**(mutex/rwsem/futex,等不到就睡,睡在 wait queue 上) vs **自旋/无锁**(spinlock/qspinlock 死等、seqlock 读者不阻塞写者、per-cpu 不竞争、RCU 读者根本不锁)。
>
> **★ 对照多本**(Tokio/Go runtime/内存分配器/Linux 调度器):锁与无锁非内核独有,Tokio 的 cmpxchg/Pin、Go 的 sync.Mutex/channel、分配器 per-CPU cache、调度器 rq->lock,处处同源。标 ★ 的章有对照栏。
>
> **基调**:直球讲透为主,比喻只在反直觉处点睛——延续《LevelDB》/Linux mm/Linux 调度器。**每个锁/RCU/内存序技巧必须讲清"为什么 sound"**(不丢更新/不死锁/不破坏内存序/不读到撕裂)。

每章一行:**一句话钩子** —— 技巧标签 —— 二分法归属(`阻塞睡眠` / `自旋/无锁` / `支撑地基` / `收束`)。

- [P1-03 · 内存屏障:为什么重排会害死你](P1-03-内存屏障-为什么重排会害死你.md) —— CPU 乱序 + 编译器重排 → 可见性/有序性崩溃 → smp_mb/rmb/wmb + smp_store_release/smp_load_acquire;Store Buffer/Invalidate Queue 直觉。 —— 经典消息传递屏障配对 —— `支撑地基`

## 第 2 篇 · 自旋锁类:不阻塞一极 ⚠️ ★对照调度器

> 等不到就自旋或根本不阻塞读者。源码主体 `kernel/locking/qspinlock.c`(queued_spin_lock_slowpath @L316 + MCS 注释 @L32)、`kernel/locking/spinlock.c`、`include/linux/spinlock.h`、`include/linux/seqlock.h`、`include/linux/seqlock_types.h`。注:`include/asm-generic/qspinlock.h`(**本地未 sparse,描述 + 引在线**)。

- [P2-05 · spinlock 演进:从 ticket 到 qspinlock](P2-05-spinlock演进-从ticket到qspinlock.md) ★ —— 朴素 TAS 公平性与缓存乒乓 → ticket lock → MCS 队列锁 → qspinlock(把 MCS 压进一个原子字)。 —— MCS 队列消灭缓存行乒乓 —— `自旋/无锁`
- [P2-06 · 顺序锁 seqlock:读者不阻塞写者](P2-06-顺序锁seqlock-读者不阻塞写者.md) ★ —— 读多写少但写者不能等(jiffies/ktime)→ seqlock 读者用奇偶版本号重读,写者加版本号前后各写。 —— 奇偶版本号 + READ_ONCE 重读 —— `自旋/无锁`
- [P2-07 · 中断禁用与 spin_lock_irqsave](P2-07-中断禁用与spin_lock_irqsave.md) ★ —— 持 spinlock 不能被中断(中断处理再拿同锁死锁)→ irqsave/bh + IRQ 上下文不可睡眠铁律。 —— IRQ 上下文不可睡眠 —— `自旋/无锁`

## 第 3 篇 · 阻塞锁:睡眠一极 ★对照 Tokio/Go

> 等不到就睡。源码主体 `kernel/locking/mutex.c`(mutex_lock @L281 + __mutex_trylock_fast cmpxchg @L171 + __mutex_lock_common @L573 + mutex_optimistic_spin @L440)、`include/linux/mutex_types.h`(struct mutex @L41)、`kernel/locking/rtmutex.c`(rt_mutex_adjust_prio_chain @L675)、`kernel/locking/rtmutex_api.c`(rt_mutex_lock @L69)、`kernel/futex/`(**本地未解压,引在线 6.9**)。

- [P3-08 · mutex:fast path + 慢路径睡眠](P3-08-mutex-fast-path与慢路径睡眠.md) ★ —— cmpxchg 抢 owner fast path + wait queue 慢路径 schedule 睡眠 + 乐观自旋 + owner 低位编码 waiters/handoff。 —— owner 低位编码 + fast/slow 分层 —— `阻塞睡眠`
- [P3-09 · rtmutex:优先级继承](P3-09-rtmutex-优先级继承.md) —— 优先级反转 → PI 链 rt_mutex_adjust_prio_chain 让持锁者继承等待者优先级;rtmutex 是 PREEMPT_RT 底层。 —— PI 链遍历 —— `阻塞睡眠`
- [P3-10 · futex:用户态锁的 fast path](P3-10-futex-用户态锁的fast-path.md) ★ —— pthread_mutex 无竞争不进内核:用户态 cmpxchg + 争用 sys_futex 进内核睡 futex_q + unlock 唤醒。 —— futex 双 fast path —— `阻塞睡眠`

## 第 4 篇 · 读写锁:读多写少场景 ★对照分配器 per-cpu

- [P4-12 · percpu-rwsem:per-cpu 计数器换读者零争用](P4-12-percpu-rwsem-per-cpu计数器换读者零争用.md) ★ —— rwsem 多读者仍 cache line 乒乓 → percpu 给每核一份 read_count,读者 this_cpu_inc 零争用,写者 rcu_sync 切换。 —— per-cpu 计数器 + rcu_sync —— `自旋/无锁`(读者) + `阻塞睡眠`(写者)

## 第 5 篇 · RCU:读者零开销的终极解 ⚠️ 核心重头戏 ★对照 Tokio/Go

> RCU(Read-Copy-Update):读者不加锁(只关抢占/禁迁移),写者复制一份改、老的延迟回收。代价是写者要等"所有读者都交卷"(宽限期)。源码主体 `kernel/rcu/tree.c`、`tree.h`、`tree_plugin.h`、`update.c`、`srcutree.c`、`sync.c`、`include/linux/rcupdate.h`、`include/linux/srcu.h`。

- [P5-13 · RCU 原理:读者无锁,写者延迟回收](P5-13-RCU原理-读者无锁-写者延迟回收.md) ★ —— rcu_read_lock 只增 preempt 计数(@rcupdate.h#L777)+ 写者复制改 + 老的等宽限期后回收(synchronize_rcu @tree.c#L3600 / call_rcu @L2836)。 —— 读者零开销为什么 sound —— `自旋/无锁`
- [P5-14 · grace period 与 quiescent state:宽限期在等什么](P5-14-grace-period与quiescent-state-宽限期在等什么.md) —— 宽限期 = 等所有 CPU 至少经过一次静默态 → tick 钩子 rcu_sched_clock_irq(@L2275)检测 → rcu_report_qs_rdp(@L2008) → rcu_do_batch(@L2119)触发回调。 —— 静默态检测 —— `自旋/无锁`
- [P5-15 · tree RCU 的层级报告:rcu_node 树](P5-15-tree-RCU的层级报告-rcu_node树.md) ★ —— 多核上"等所有 CPU 静默"怎么不慢:rcu_node 层级树(@tree.h#L41)+ GP 状态机(@L411-L419)+ rcu_gp_kthread(@tree.c#L1836)+ rcu_gp_init/cleanup。 —— rcu_node 层级树 + 状态机 —— `自旋/无锁`
- [P5-16 · srcu:可睡眠的 RCU 读者](P5-16-srcu-可睡眠的RCU读者.md) —— 普通 RCU 读者不能睡;srcu 用双计数槽让读者可睡(@srcutree.c#L712/__srcu_read_lock,@L989/srcu_funnel_gp_start)。 —— srcu 双计数槽 —— `自旋/无锁`
- [P5-17 · RCU 同步原语家族:rcu_sync 与 Tasks Trace](P5-17-RCU同步原语家族-rcu_sync与Tasks-Trace.md) —— 基于 RCU 的轻量原语:rcu_sync(kernel/rcu/sync.c)被 percpu-rwsem 用 + Tasks Trace RCU 用于 Tracing。 —— rcu_sync 状态机 —— `自旋/无锁`
