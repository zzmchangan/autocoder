# 全书导言 · 一篇看懂 Linux 内核全貌

> 这篇是整本书的"正门"。读任何一篇之前先读它,你会对内核是个什么东西、每个部门在干嘛、为什么必须存在,建立一个结实的直觉。
>
> 全文只讲两件事:**内核在干什么**,以及**为什么非得这么干**。
> 我会用一个贯穿全书的比喻——**一台机器 = 一座城市,内核 = 这座城市的市政府**——帮你记住每一个机制。

---

## 一、一句话说清内核在干嘛

> **内核是一座城市的市政府。它干的所有事,就是让这座城市里每一个市民(进程)都觉得自己是这座城市唯一的主人,而实际上几千个市民挤在同一座城里、抢着用同一套公共设施,居然不打架、不崩溃、还挺快。**

这就是内核存在的全部理由。后面所有机制——内存、进程、调度、中断、文件、设备——都是这个市政府的各个职能部门,各自解决"如果没人管就会乱套"的一类问题。

你任何时候看不懂某个机制,就回来问自己一个问题:

> **"这个机制,是在'管好大家共享的公共资源',还是在'给某个市民制造独占的幻觉'?"**

整个内核也就这两大块活儿。记住这个二分法,你就永远不会迷路:

- **管资源那一侧**:CPU 时间、内存、磁盘、设备——这些是全城共享的、有限的、谁都想多占的硬通货。内核负责公平、高效地分配它们。
- **造幻觉那一侧**:让每个市民都以为"这台电脑是我一个人的"——我有用不完的 CPU、有独占的内存、有一套好用的文件。这套幻觉,是市民能安心写程序的前提。

内存篇你已经见过这个二分法("物理侧切房间 / 虚拟侧造幻觉")。现在把它放大到全内核:**"管共享资源"是物理侧,"造独占幻觉"是虚拟侧**——所有子系统都在这两侧之一,或两边都沾。

---

## 二、建立比喻:一台机器 = 一座城市

为什么不用"管家""导演""乐队指挥"那些比喻?因为它们都太小,装不下内核这么多部门。**城市/市政府**是少数能同时容纳"资源管理 + 幻觉制造 + 多部门协作"的大比喻。

下面这张表是**全书术语统一的总开关**。每遇到一个新概念,先回来查它在这张表里的"户口":

| 城市比喻 | 内核概念 | 一句话 |
|----------|----------|--------|
| 整座城市 | 整台机器 | 全书的舞台 |
| **市政府** | **内核** | 站在所有市民之上、管公共事务的政府 |
| 市民 | **进程** | 一个正在运行的程序 |
| 市民的分身 | **线程** | 同一个市民,同时干多件事 |
| 办事窗口 / 劳动力 | **CPU** | 全城最稀缺的资源,谁此刻能用 |
| 办事大厅叫号 | **调度器** | 决定哪个市民此刻占用窗口 |
| 土地与房屋管理局 | **内存管理** | 管城市里所有的房子(内存篇专讲) |
| 一栋大楼 / 一间房 | RAM / 一个物理页 | 房屋管理局管的最小单位 |
| 市民去政府办事的窗口 | **系统调用** | 市民不能自己动手,必须排队申请 |
| 紧急呼叫 / 门铃 | **中断** | 外部硬件主动找政府 |
| 报警 / 闯祸 | **异常** | 市民自己出错(除零、缺页),惊动政府 |
| 城市钟楼 | **时钟中断** | 全城的心跳,驱动调度和定时 |
| 多部门同时改一份档案 | **并发与同步** | 锁、信号量、RCU |
| 档案馆 / 图书馆 | **文件系统(VFS)** | 数据怎么存、怎么找、怎么不丢 |
| 档案馆的地下仓库 | **磁盘 / 块设备** | 真正存东西的地方 |
| 市政公用事业(水电气) | **设备与驱动** | 连接外部世界 |
| 邮政 / 电信 | **网络协议栈** | 和其他城市通信(进阶) |
| 把城市划成独立小区 | **namespace** | 小区之间互相看不见(容器) |
| 给小区配额(限水限电) | **cgroup** | 限制某个小区能用多少资源(容器) |

这张表先扫一眼留个印象。下面我们**跟着一座城市从一片漆黑到灯火通明**走一遍,每经过一个部门,就回来对应这张表。

## 三、跟着旅程走一遍:从按下电源键到 `ls` 输出结果

整本书的主线,就是这段旅程。每个子系统,都是旅程上的一个驿站。

### 第 0 站:一片漆黑——城市还没通电

你按下电源键。此刻 CPU 处在最笨的状态(只认识最低 1MB 地址),操作系统还躺在硬盘里。

> 真正在管事的是 **BIOS/UEFI**——相当于"建城施工方"。它干两件事:**清点这座城市有多少房子、哪些被设备占了**(写成一张"满是窟窿的内存地图" e820);然后把内核这个"市政府"请进城。

内核醒来时,手里只有施工方递来的一张破破烂烂的城市地图。

### 第 1 站:市政府开张(启动 / Booting)

内核(市政府)醒了,可自己连办公室都没有。它得先**把自己撑起来**:

- 用临时材料搭个工棚(临时页表),让自己能正常运转;
- 因为正式的资源分配部门(房屋管理局=伙伴系统)还没就位,先派个**临时工(memblock)**记两本粗账,熬过启动期;
- 顺着一条叫 `start_kernel` 的初始化大游行,把所有部门一个个挂起来;
- 最后,**第一个市民诞生**——0 号进程,它是所有市民的祖先。

> 到这一站结束,市政府挂牌营业,但城里还只有一个市民。

### 第 2 站:房屋土地管理局(内存管理)

这是本书**已经写得最完整的部门**(见内存篇)。它的活儿分两面:

- **管资源**:把全城的房子(物理内存)切成 4KB 一间,按地段分区(ZONE),用伙伴系统分配回收,再用 slab 把房子切成更小的预制件给内核对象用。
- **造幻觉**:给每个市民发一本**私家门牌本(虚拟内存)**,让他以为自己独占了全城所有房子;他敲哪扇门,房屋管理局才临时给他配一间真房(page fault 兑现)。

> 这是"管资源 + 造幻觉"二分法最完整的展示。后面所有部门,都是这两个主题的变奏。

### 第 3 站:市民与户籍科(进程)

内存有了,可谁来住?**进程**就是市民。户籍科(进程管理)负责:

- 给每个市民建一份档案(`task_struct`),记录他是谁、住哪、在干嘛;
- 市民的**出生**:`fork` 一个新市民(几乎不复制房子,靠"写时复制");`exec` 让他换一身新衣服(加载新程序);
- 市民的**死亡**:`exit`,把房子退回房屋管理局。

一个市民从生到死,全在这一站。

### 第 4 站:办事大厅与叫号器(调度)

城里有几千个市民,可办事窗口(CPU)只有几个(几个核)。谁此刻能用?

**调度器**就是那个叫号器。它要让"每个市民都觉得自己独占了窗口"——办法是**极快地轮换**(每秒上千次),每个市民用一小片时间(vruntime),快到市民根本感觉不到自己被换下来过。

> 这就是内核造的**第一个幻觉:独占 CPU**。实际上你从来没独占过,只是调度器骗了你。

### 第 5 站:市政府的门与门铃(中断、异常、系统调用)

市民平时在自己的世界里跑,什么时候需要市政府介入?三种情况,合称"陷入内核":

- **系统调用**:市民**主动**来窗口办事("请帮我读个文件""请帮我建个市民")。市民不能自己动手碰公共资源,必须通过窗口申请——这是**安全**和**隔离**的关键。
- **中断**:外部硬件**主动**拍市政府的肩膀("键盘有输入了""网卡收到包了""时钟又走了一格")。门铃响了,政府得去处理。
- **异常**:市民**自己闯祸**(除以零、访问了不该访问的地址),CPU 自动报警,政府介入(通常是教训甚至开除这个市民)。

> 这三种本质上都是"从市民的世界切换到政府的世界"。这一站讲透了"内核什么时候、为什么会突然插手"。

### 第 6 站:档案馆(文件系统)

市民的资料要存起来、要能找到、掉电不能丢。**文件系统(VFS)**就是档案馆:

- 它给所有市民一个**统一的幻觉**:"世界上一切都是文件"——硬盘是文件、键盘是文件、网卡是文件。不管底下是什么,市民都用同一套"打开/读/写/关闭"的套路。这是内核造的**第二个幻觉:统一的文件世界**。
- 真正的资料躺在**地下仓库(磁盘/块设备)**里。档案馆负责把"市民想读第几页"翻译成"去仓库哪个货架取",还要排队、合并、调度,让仓库别被挤爆。

### 第 7 站:公用事业科(设备与驱动)

城市要通水通电通气,得有人**认识并驱动**各种外部设备。设备模型(`kobject`/`sysfs`)就是市政府的"设备登记处",驱动就是每种设备的"操作手册"。

### 终点:`ls` 输出结果

走完全程,你现在能完整地解释一行 `ls` 是怎么跑起来的:

> 你在终端敲下 `ls` → shell **fork** 出一个新市民,再 **exec** 换上 `ls` 程序 → `ls` 这个市民想列出当前目录,于是发起**系统调用**(去办事窗口) → 内核接管,走到**文件系统(档案馆)** → 档案馆去**磁盘(地下仓库)**取目录数据 → 数据可能已经在**内存(page cache)**里,直接返回 → 内核把结果通过系统调用**交还**给 `ls` 市民 → `ls` 调系统调用**写**到屏幕 → 你看到了文件列表。

这一行命令,**经过了几乎所有部门**。这就是为什么"从上电到 `ls`"是理解内核最好的骨架——它逼你把每个部门都串起来,而不是孤立地背。

## 四、收束:内核的"三大幻觉" + 两句话

走完全城,你会发现内核这个市政府,毕生精力都花在**卖三个幻觉**上:

1. **独占 CPU 的幻觉**——你从没独占过,调度器在飞速轮换(进程/调度篇)。
2. **独占内存的幻觉**——你从没独占过,虚拟内存让你以为全城房子都是你的(内存篇)。
3. **统一的文件/设备世界的幻觉**——硬盘、键盘、网卡千差万别,VFS 让你都觉得是"文件"(文件篇)。

每个进程都活在这三个幻觉里,安心地写着程序,完全不知道底下市政府忙成什么样。**这三个幻觉的存在,就是内核存在的意义。**

### 记住这两句话,就抓住了全书

1. **内核一半在"管共享资源",一半在"造独占幻觉"。** 任何时候迷路,回到这个二分法,问"这站在管资源还是造幻觉"。
2. **所有复杂机制都是被三个现实逼出来的**:资源有限(不够分)、大家共享(会冲突)、硬件各异(要统一)。当你觉得某个设计"为什么搞这么复杂",去找是哪个现实在作怪。
