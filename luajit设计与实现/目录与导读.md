# 《LuaJIT设计与实现深入浅出》·目录与导读

> 主线:**把动态执行安全变成机器码**(trace + guard + side exit)。二分法:**解释器侧 ↔ JIT 侧**。★对照:**官方 Lua + JVM/V8**。源码:**LuaJIT 2.1.ROLLING**。基调:**纯直球,不用比喻**。

---

## P0 开篇

- **P0-01 第一性原理:JIT 凭什么这么快——trace compiler 是什么**
  提出全书主线。讲清 JIT 的核心张力(快/安全/省)与 LuaJIT 的解法:trace(只录热路径)+ guard(假设编进机器码)+ side exit(失败回解释器)。对比官方 Lua(纯解释器)和 JVM/V8(method JIT)。

## P1 解释器与热点(JIT 的基座)

- **P1-02 字节码与手写汇编 VM**
  LuaJIT 的字节码(`lj_bc`)与栈式解释器(`lj_vm.s` 手写汇编)。为什么手写汇编(每条指令的 dispatch 都要省到极致)。★对照官方 Lua(寄存器式字节码)。
- **P1-03 dispatch 表与热点检测**
  `lj_dispatch` 的全局函数表 + `hotcount` 计数器(`HOTCOUNT_SIZE=64`,pc>>2 哈希)。怎么发现热点、触发 trace 录制。
- **P1-04 值·Table·状态**
  `lj_obj`/`lj_tab`/`lj_state`。LuaJIT 与官方 Lua 在值表示/Table 上的异同(LuaJIT 有 NaN-boxing、tagged)。★对照官方 Lua。

## P2 trace 录制与 IR(全书核心·上)

- **P2-05 trace 的生命周期**
  `lj_trace`:一个 trace 从热点触发→录制→优化→生成机器码→安装→运行→退出→链接的完整一生。`GCtrace` 结构。
- **P2-06 录制:字节码变 IR(record)**
  `lj_record`(2940 行):边执行边把字节码翻译成 IR,遇到分支/guard 就停下。录制的主循环。
- **P2-07 IR 与 SSA**
  `lj_ir`:`IRIns`(union op1/op2/prev)、IRRef、SSA 构建、REF_BIAS 偏移编码。LuaJIT 的线性 SSA。
- **P2-08 类型窄化 narrowing**
  `lj_opt_narrow`:把动态类型(number)推成具体类型(int/double),这是动态语言能 JIT 的关键。★对照 V8 hidden class。

## P3 优化 pass

- **P3-09 常量折叠 fold**
  `lj_opt_fold`(2655 行,最大 pass):代数化简、常量传播、强度削减。基于 IR 规则表。
- **P3-10 内存优化与别名分析**
  `lj_opt_mem`:load/store 消除、别名分析(两个引用是否指向同一对象)。
- **P3-11 分配消除 sink + 死代码 dce**
  `lj_opt_sink`(短命分配消除)+ `lj_opt_dce`(死代码消除)。
- **P3-12 循环优化 + 指令拆分**
  `lj_opt_loop`(循环不变量外提)+ `lj_opt_split`(把双精度指令拆成可单精度优化)。

## P4 后端:IR 到机器码

- **P4-13 线性扫描寄存器分配**
  `lj_asm` 里的寄存器分配:线性扫描(非图着色),快且够用。为什么 trace 适合线性扫描。
- **P4-14 汇编生成 asm**
  `lj_asm`(2643 行):遍历 IR,逐条发射机器码。guard 检查、spill、fixup。
- **P4-15 后端目标与机器码内存**
  `lj_target_*`/`lj_emit_*`(x86/ARM 多后端)+ `lj_mcode`(可执行内存分配,W^X 保护)+ dynasm。

## P5 运行时:guard、snapshot、side trace

- **P5-16 guard 运行时检查与 side exit**
  生成的机器码里的 guard 检查怎么执行、失败怎么跳到退出处理(`lj_vm.s` 的 exit stub)。
- **P5-17 snapshot 恢复解释器状态**
  `lj_snap`:每个 guard 一个 snapshot,退出时把寄存器/栈状态恢复成解释器能继续的样子。
- **P5-18 side trace:从退出点录制新 trace**
  某个 guard 经常失败,就从那个退出点录制一条新的 side trace(trace 树的 child)。
- **P5-19 trace 链接**
  `TraceLink` 9 种:loop(回自己)/tailrec/uprec/downrec/stitch(缝合)/interp(退回解释器)。trace 之间如何织成网。

## P6 FFI 与 GC

- **P6-20 FFI:C 类型与调用**
  `lj_ctype`(C 类型系统)+ `lj_ccall`(C 函数调用):LuaJIT 能直接调 C,不必走 Lua C API。
- **P6-21 FFI 录制 + GC 与 JIT 协作**
  `lj_crecord`(FFI 调用也进 trace 录制,JIT 调 C!)+ `lj_gc`:GC 与 JIT 的协作(snapshot 要记录 GC 状态、mcode 也要被 GC 管理)。
