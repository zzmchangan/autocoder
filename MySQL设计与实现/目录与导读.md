# 《MySQL·InnoDB 设计与实现深入浅出:一次写怎么做到既不丢又高并发》—— 目录与导读

> 一本写给"天天用 MySQL、调过索引、背过隔离级别,却总觉得 InnoDB 是个黑盒"的人的小书。
>
> **一句话主旨**:一条写,InnoDB 用四个保证让它"既不丢又高并发":B+树聚簇索引找到位置、redo(WAL)保 crash 不丢、undo(MVCC)保并发读、锁保隔离。
>
> **二分法**(迷路时回到它):**存储与索引**(数据怎么放、怎么找:B+树聚簇索引/buffer pool/doublewrite) vs **事务与并发**(怎么不丢、怎么并发:WAL/redo/undo/2PC + MVCC + 锁)。
>
> **主比喻**:直球为主、比喻点睛。开篇用"图书馆"做一次性点睛(聚簇索引=按学号排的书架;二级索引=按姓名查学号再回书架;WAL=先记账本再搬书),全书不再建立贯穿重比喻。
>
> **四重承接**:聚簇索引↔《PG》堆表、B+树↔《LevelDB》LSM、buffer pool↔《Linux内存管理》page cache、MVCC/事务↔《TiKV》分布式事务。

每章一行:**一句话钩子** —— 技巧标签 —— 二分法归属(`存储与索引` / `事务与并发` / `衔接` / `总览`)。

旅程:从"MySQL 怎么存一行数据",一路拆到"一条写怎么在 crash 不丢、并发不乱的情况下落盘"。每篇都是这条旅程上的一个驿站——读完你能在脑子里放映出一条 `UPDATE` 在 InnoDB 里的全过程:server 层解析优化 → handler 进 InnoDB → buffer pool 找页 → 加锁 → 改数据 + 写 undo → 生成 redo(mtr)→ 两阶段提交(redo prepare→binlog→redo commit)→ 脏页经 doublewrite 刷盘 → crash 用 redo 重放 + undo 回滚。
