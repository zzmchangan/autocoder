- `AE_BARRIER`(**不译**);`AE_DELETED_EVENT_ID`(**不译**)

- **skiplist 跳表**(**勿**写"跳跃表")

- **span 跨度**(= 该 forward 指针跨过的底层节点数)

## 四、全书通用纪律(所有 agent 必须遵守)

1. **五条设计取向**统一编号与措辞:①单线程 + 事件循环 ②内存即数据库 ③编码自适应 ④简单优先 ⑤可靠性靠持久化 + 复制。每章回扣"本章落在取向 X",诚实标注,不强求一致。
2. **验证物三段**统一结构:① gdb 断点脚本(对应真实行号)② 源码常量锚点表(带行号)③ OBJECT ENCODING 观察项(预期输出配推导链)。开头统一标注:"本书 Windows 环境未实跑,以下供 Linux `make no-opt` 编译后复现,不附编造输出。"
3. **钉死句式**统一:`> **钉死这件事**:……`(每章 8-12 次)。
4. **全角标点**(代码块、行号、变量名、URL 除外)。
5. **行号必须 Grep/Read 核实**后引用,不符停下报告主控,**不得凭记忆编造**。Redis 8.0.2 与旧版(7.x/6.x)行号差异大。
6. 引用格式:`[描述](../../redis-8.0.2/src/文件名#L行号)`(相对 redis设计与实现/ 目录)。
7. **逃生阀**只给高难度章(ae/dict/skiplist/RDB/Cluster/AOF/复制 等);简单章(SDS/listpack)不给。
8. **图**:流程/时序用 mermaid;内存/数据结构布局用 ASCII 框图;**勿**用 ASCII 画时序。
