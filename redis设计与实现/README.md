# 《Redis 设计与实现深入浅出》

> 源码:**Redis 8.0.2**(2025 GA,这一版 Redis 重回 BSD 开源许可)
> 系列定位:"源码精解大书"第四本,与同系列《Tokio 设计与实现深入浅出》对仗——一本讲运行时怎么解放"等待",一本讲服务端怎么解放"耗时",并把整个设计讲清。

---

## 本书主轴

> 不为单句口号所囿,把 Redis 的每个设计决策讲清楚——**为什么这样、不那样**,动机(why)+ 精妙(how)双线,并讲清它换来了什么、付出了什么代价。

**五条设计取向**(每章回扣其中一条或几条,诚实优先,不强求一致):

1. **单线程 + 事件循环**——无锁、无并发 bug、可预测。把耗时从主线程解放出去的那一连串招(渐进 rehash、懒删除、fork 子进程、后台线程、IO 多线程)。
2. **内存即数据库**——一切常驻内存,为每种访问模式定制数据结构换性能。
3. **编码自适应**——一套命令接口,多种底层编码,按数据规模/形态自动切换。
4. **简单优先**——能用简单结构就不用复杂的(skiplist 不用红黑树、SDS 不用 std::string、自造而非引依赖)。
5. **可靠性靠持久化 + 复制**——RDB/AOF 兜底持久化,主从 + Cluster 兜底高可用。

## 写作说明(关于准确性)

本书所有源码引用均基于 **Redis 8.0.2**,写作中逐章 Grep/Read 核实行号,并据实修正了若干"旧版教程/凭印象"会写错的地方(详见[启动包](00-启动包.md)):

- **dict 结构 8.0 拆平**:旧版嵌套的 `dictht` 没了,字段直接铺进 `dict`(`ht_table[2]`/`ht_used[2]`/`ht_size_exp[2]`)。
- **listpack 取代 ziplist**:7.0 起 listpack 根治了 ziplist 的连锁更新问题。
- **MP-AOF**:8.0.2 的 AOF 是目录形态(base + incr + manifest),rewrite 不再用 `aof_rewrite_buf`。
- **repl_backlog 链表化**:8.0.2 的复制积压缓冲是 `replBufBlock` 链表 + rax + 引用计数(旧版是环形缓冲)。
- **IO 多线程 8.0 重构**:每 IO 线程有独立 `ae` 循环,**协议解析也下放到了 IO 线程**,只有命令执行留在主线程。
- **keymeta 在 8.0.2 实为 mstr**:8.0.2 还没有 `keymeta.c`(那是 8.4+),真正的"key 元信息抽象"雏形是 `mstr`,目前仅用于 hash field TTL。

> 配套源码:本地目录 `../redis-8.0.2/`(已 clone,version.h 核实为 `8.0.2`)。
