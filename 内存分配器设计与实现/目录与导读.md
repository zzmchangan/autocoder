---

## 这本书讲什么(一句话)

> **`malloc`/`free` 是一切程序的底座。操作系统给的 `mmap`/`brk` 太粗,glibc ptmalloc 的 arena 锁在高并发下成瓶颈、碎片也压不住。tcmalloc / jemalloc / mimalloc 这一代分配器,把"一次 malloc"拆成三层快慢道——线程本地缓存(无锁,O(1))→ 中心自由链表(按大小分级,批量)→ OS 页堆(整批向系统要/还内存)。本书沿着这条旅程,把四套分配器并排对照讲透:每个机制**为什么**这么设计、源码里那个原子操作 / per-CPU / radix tree / `madvise` **怎么**做到的、不这么做会撞什么墙。**

## 怎么读这本书

### 一条主线、一个二分法,随时回到它们

- **主线**:如何让 `malloc`/`free` **又快又省**?快(无锁 fast path、低争用)与省(低碎片、低占用、及时归还 OS)是永恒张力。
- **二分法**:**局部缓存**(线程/CPU 私有,无锁 fast path,要"快")vs **中心堆**(全局共享,slow path,管页/合并/归还,要"省")。
- 任何时候看不懂某个机制在干嘛,回到二分法问一句:"这是在让线程本地分配更快,还是在让全局内存更省更整齐?"

### 建议路线

- **第一次读**:按章序从头读到尾(P0 → P7)。旅程是一条线,前一章的地基是后一章的前提。
- **只想搞懂性能瓶颈**:重点读 **第 1 篇(三层快慢道)+ 第 3 篇(多核并发)**,这是"快"的核心。
- **被内存占用 / 碎片折磨**:跳到 **第 4 篇(碎片治理与归还)**,尤其第 15 章 huge page aware allocator。
- **做调优 / 抓泄漏**:看 **第 18 章(profiling)+ 附录 B(实战)**。
- **想快速建立全局观**:先读 **第 1 章(P0-01)**和 **第 20 章(四套对照总表)**,再回头细读。

### 难度标记

- ⭐ 入门:概念、动机,少源码。
- ⭐⭐ 进阶:有真实源码与技巧精解。
- ⭐⭐⭐ 硬核:rseq、huge page filler、radix tree 等,有逃生阀,先抓主旨。

## 四套源码,怎么对照

本书是**横评**:每个机制都四套并排看。它们的角色:

| 分配器 | 角色 | 本地路径 / 版本 | 特点一句话 |
|--------|------|-----------------|-----------|
| **tcmalloc** | 主角① | `../tcmalloc/` @ `7723f74`(Google 新版 C++) | per-CPU cache + HPAA,工业最新 |
| **jemalloc** | 主角② | `../jemalloc/` @ `9f37c70` | 多 arena + radix tree + decay purge,Facebook 主力 |
| **mimalloc** | 新秀对照 | `../mimalloc/` @ `fef6b0d` | segment/page 两层 + 随机化 + arena-abandon,微软新设计 |
| **ptmalloc** | baseline | 在线 `malloc/malloc.c` | glibc 默认,arena+chunk+bins,"朴素方案"反衬 |

正文里引用格式:本地三套 `[描述](../tcmalloc/...#L起-L止)` 等;ptmalloc 在线 `[描述](https://github.com/glibc/glibc/blob/main/malloc/malloc.c#L起)`。

## 全书目录

### 第 0 篇 · 开篇:为什么自己写 malloc(1 章)

- **第 1 章 · 第一性原理:为什么 glibc malloc 不够用**(`P0-01`)⭐
  OS 的 `mmap`/`brk` 太粗、ptmalloc 的 arena 锁与碎片 → 三层快慢道的直觉 → "快 vs 省"张力 → 立"局部缓存 vs 中心堆"二分法 → 四套亮相。

### 第 1 篇 · 共通地基:三层快慢道与 size class(5 章)

- **第 2 章 · size class:把任意大小凑整成分级**(`P1-02`)⭐⭐
  为什么不按原样分配、归并成几十个分级、内部碎片 vs 查表 O(1)。四套 size class 对照。技巧:size→class 位运算映射。
- **第 3 章 · 对齐与缓存行:size class 为什么这么对齐**(`P1-03`)⭐⭐
  对齐到 8/16/缓存行/pow2 的原因、防 false sharing。技巧:缓存行对齐。
- **第 4 章 · 自由链表:用释放块本身串链**(`P1-04`)⭐⭐
  `next` 藏进释放块、push/pop O(1)、LIFO vs FIFO。四套 free list 对照。技巧:无锁单线程 + 跨线程原子 push。
- **第 5 章 · 线程本地缓存:fast path 必须无锁**(`P1-05`)⭐⭐
  为什么 fast path 必须无锁、TLS、cache 满/空。四套 tcache 对照。技巧:TLS 的几种实现与懒创建。
- **第 6 章 · 中心自由链表与 transfer cache:cache miss 后的批量取还**(`P1-06`)⭐⭐
  miss 后去中心批量补货、transfer cache 线程间流转。四套对照。技巧:批量平摊锁开销。

### 第 2 篇 · 页堆:批量向系统要内存(3 章)

- **第 7 章 · 页、span、extent:内存按页管理**(`P2-07`)⭐⭐
  中心链表空了去页堆、span/extent/segment 建模。技巧:span 连续页+长度 O(1) 切并。
- **第 8 章 · pagemap 与 rtree:指针 → 所属 span 的 O(1) 反查**(`P2-08`)⭐⭐⭐
  `free(p)` 怎么 O(1) 找到归属。技巧:放射 pagemap vs radix tree 取舍。
- **第 9 章 · 大块分配:huge/large 的单独路径**(`P2-09`)⭐⭐
  超大块直接 mmap、不走 size class。四套 huge/large 对照。技巧:mmap threshold 自适应。

### 第 3 篇 · 多核并发:不让锁成瓶颈(3 章)

- **第 10 章 · 争用从哪来、三种解法**(`P3-10`)⭐⭐
  锁争用根源 + 三种解法(多 arena / per-CPU / 无锁)总览。
- **第 11 章 · jemalloc 的 arena 与 bin**(`P3-11`)⭐⭐⭐
  4×ncpus arena 分流、每 bin 一把锁。技巧:arena 数量与线程绑定的权衡。
- **第 12 章 · tcmalloc 的 per-CPU cache**(`P3-12`)⭐⭐⭐
  按 CPU 核缓存、rseq 安全回退。技巧:rseq + per-CPU slab(tcmalloc 相对 jemalloc 的代差)。

### 第 4 篇 · 碎片治理与内存归还(3 章)

- **第 13 章 · 碎片的两个来源与合并**(`P4-13`)⭐⭐
