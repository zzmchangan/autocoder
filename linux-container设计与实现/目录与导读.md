# 《Linux 容器与命名空间深入浅出:cgroup v2 + namespace 怎么造出容器》—— 目录与导读

> 一本写给"用过 Docker / K8s、甚至翻过 `kernel/cgroup/` 和 `kernel/nsproxy.c`,却总觉得一知半解"的人的小书。
>
> **一句话主旨**:容器 = 一个被关进"看起来独占整机"盒子的进程。namespace 改它的视图(看不见别人),cgroup 限它的资源(吃不到别人),两者一夹,就是容器。
>
> **二分法**(迷路时回到它):**资源控制(cgroup:cpu/memory/io/pids/freeze 限额)** vs **视图隔离(namespace:mnt/pid/net/uts/ipc/user 看到什么)**。
>
> **★ 对照 runc / Docker / K8s**:内核提供积木(namespace + cgroup),用户态运行时(runc 按 OCI 规范)把它们组装成容器。标 ★ 的章有对照栏。
>
> **基调**:直球讲透为主,比喻只在反直觉处点睛——延续《LevelDB》/《Linux mm》/《Linux 调度器》。

每章一行:**一句话钩子** —— 技巧标签 —— 二分法归属(`视图` / `资源` / `组装` / `收束`)。
