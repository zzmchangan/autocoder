# 核心比喻:橡皮膜(全书统一,不得跑偏)

全书统一用**橡皮膜**比喻,术语必须落在总纲的对照表内。核心对应(背下来):

- **橡皮膜上一根从原点出发的箭头** = 向量。
- **用几份基准箭头调配出别的箭头** = 线性组合(像调色)。
- **几根箭头能调配出的全部箭头** = 张成(span)。
- **调配时不冗余的最少几根基准** = 线性无关 / 基 / 维数。
- **伸手揉这张膜(拉/转/压扁/剪切,原点不动、网格平行等距)** = 线性变换。
- **记录"基准箭头被揉去哪"的说明书** = 矩阵(每一列=一根基向量的新位置)。
- **两次揉捏接龙** = 矩阵乘法(先右后左,不可交换)。
- **揉捏后面积/体积胀缩的倍数** = 行列式;**揉捏把膜压扁了(面积变0)** = 行列式=0/不可逆。
- **揉捏的"撤销键"** = 逆矩阵。
- **揉捏后空间还剩几维** = 秩;**揉捏中被压扁到原点的箭头** = 零空间。
- **揉捏中方向不变、只被拉长/压短的轴** = 特征向量;**那根轴被拉伸的倍数** = 特征值。
- **换一套基准箭头重新描述同一个揉捏** = 基变换 / 相似矩阵。
- **把任意揉捏拆成"转→拉→转"三步** = SVD。

新概念先给橡皮膜比喻,再落到算式。可以叠加子比喻(如"调色""骨架""团队冗余"),但**不得与橡皮膜体系冲突**,更不得另起一套(如城市、人体、动物等——那是别的系列的家当)。

# 核心写作原则(最重要,反复对照)

1. **"为什么"永远先于"怎么做"。** 全书最高原则。每个概念都要先回答:
   - **它在几何上描述什么?**(先用橡皮膜讲清这个动作/侧面)
   - **不这样理解会怎样?**(只会算的人会卡在哪、踩什么坑)
   - **所以这样看。**(算式是这个几何的速记,是理解的副产品)
   - 然后才落到公式/算法。
2. **几何直觉优先,公式是副产品。** 任何算式(行乘列、行列式展开、特征方程)都必须**先有几何,再有公式**,且最好能让读者自己"推出"公式。严禁上来就甩公式。
3. **大白话 + 橡皮膜比喻。** 落在对照表内。
4. **小白友好,先卸包袱。** 不假设读者记得教材定义。任何术语首次出现一句话点破。开篇类章节要明说"不懂不是你的错,是讲法反了"——这是对读者最重要的承诺。
5. **每章配真实几何图 + 计算佐证。** 见下方"配图工作流"和"计算佐证"。
6. **回扣全局。** 每章结尾回到"**代数 vs 几何**"二分法,说明本章概念是"揉捏"的哪个侧面,衔接前后章节。

# 深度方针:越深越好,但"由浅入深、深入浅出"

读者明确要求:**内容要尽量深(含推导、进阶、函数空间、SVD 等巅峰内容),但路径必须由浅入深、最终能浅出。** 写每一章都要自检三件事:

- **浅入口**:从最直观的地方起步(一个生活比喻 / 一根箭头 / 一个反直觉的提问),别一上来就硬核。
- **深落点**:在直觉之上,给出真正有分量的深度——公理层面的"为什么"、反例、函数空间的推广、与前序概念的暗线勾连。**不要停在科普。**
- **浅出收束**:章末用一句话或一个比喻把全章浓缩成可记忆的 nugget,让深的也能被带走。

**彩蛋机制(兑现"越深越好")**:在合适的章节,加一节"函数空间"彩蛋——把该章概念推广到函数(多项式、傅里叶、泰勒),让读者看到线代是"几何/代数/分析"三界的通用语言。P1-02、P1-03 已有范本,后续秩/特征值/SVD 等章可继续。

# 每章的标准结构(严格照此组织,顺序不可变)

1. **章首**:
   - **核心问题**(一句话)+ "读完本章你会明白……"(3~4 条,每条戳一个"为什么")。
   - 一段简短引子,把上一章的线索接过来。
   - 难度高、信息密度大的章节,额外加**逃生阀**:`> **如果一读觉得太难**:先只记住三件事——①…;②…;③…。`
2. **章首·一句话点破**:给一个核心比喻/金句(结论),然后说"这句话是结论,不是理由;本章倒过来拆"。
3. **正文·若干小节**:每节遵循 **"提出问题 → 不这样会怎样 → 所以这样看 → (计算佐证/配图)"** 四段式,可用小标题分割。**招牌句式**反复用:
   - `> **比喻**:……`(橡皮膜/子比喻)
   - `> **不这样(看/理解)会怎样**:……`
   - `> **所以这样看**:……`
   - `> **钉死这件事**:……`(关键结论用这个收口)
4. **计算佐证**(正文之后、小结之前):给 2~3 个**纸笔就能算**的例子 + 一段 **numpy** 代码,亲手验证"比喻 = 算式"。这是数学书的"代码佐证",每章必有。
5. **章末小结**:
   - 用橡皮膜比喻回顾本章;
   - 回扣全书主线(代数 vs 几何;本章是"揉捏"的哪个侧面);
   - **五个"为什么"清单**(读者若只记五件事);
   - "想继续深入该往哪钻"(3Blue1Brown 对应集、numpy 实验、函数空间尝鲜);
   - 一句话衔接、引出下一章。

# 配图工作流(线代特色 —— 图就是它的"可验证物")

线代是高度几何化的学科,**每章必须配真实几何图**(向量、变换、网格揉捏前后、面积变化等),**严禁只用 ASCII 图**(读者明确嫌弃)。流程如下:

1. **共享工具**:所有绘图都 `import` 自 `线性代数入门/figures/_plot_utils.py`(提供 `arrow` / `grid` / `frame` / `save` 和统一配色 `GREEN/RED/BLUE/PURPLE`)。**先读这个文件,了解可用工具。**
2. **独立脚本,绝不碰共享文件**:为你的章节写一个**独立**脚本 `figures/fig-{篇}-{章}-gen.py`(例:`fig-3-09-gen.py`)。脚本顶部:
   ```python
   import sys, os
   HERE = os.path.dirname(os.path.abspath(__file__))
   sys.path.insert(0, HERE)
   from _plot_utils import arrow, grid, frame, save, GREEN, RED, BLUE, PURPLE
   import numpy as np, matplotlib
   matplotlib.use("Agg")
   import matplotlib.pyplot as plt
   ```
   **严禁修改 `make_figures.py`(那是第 1~3 章的生成器)和 `_plot_utils.py`(共享,并行安全靠它只读)。**
3. **图内标注一律用英文**(避免中文字体乱码),正文用中文。
4. **运行生成**:`python figures/fig-{篇}-{章}-gen.py`,产出 `figures/fig-{篇}-{章}-{序}-{主题}.png`。
5. **嵌入章节**:用 `<img>` 标签控宽(markdown 原生 `![]` 控不了尺寸,会撑满):2×2 组图 `width="560"`,单图 `width="420"`。例:
   ```
   <img src="figures/fig-3-09-1-area-scale.png" width="560" alt="图 9.1 · 行列式 = 揉捏后面积的缩放比">
   ```
6. **图必须画对几何**:所有向量/矩阵的坐标,先用 numpy 算对再画;行列式面积、特征向量方向等,要和正文数字一致。**画错几何是线代配图最致命的硬伤。**

> 配图建议数量:基础章 1~2 张;几何密集的章(行列式、特征值、SVD)2~3 张。每张图前用一句引导文字告诉读者"看什么"。

# 写前必做:几何与计算核查(防止画错 / 算错,这一步不能省)

数学书没有源码,但有同样致命的硬伤——**几何画错、举例算错、公式与几何对不上**。引用 / 举例前必须核实:

1. **每个举例的矩阵 / 向量 / 行列式 / 特征值,先用 numpy 算对**(尤其行列式、`np.linalg.eig`、`matrix_rank`、`inv`),再写进正文。**严禁凭印象写数字。**
2. **每张配图的坐标,必须和正文数字一致**;变换前后的网格、箭头方向、面积,要经得起核对。
3. **每个公式,都要能追到一句"几何上它是什么"**;如果一个公式你没法用橡皮膜解释,说明你还没懂透,先想清楚再写。
4. **不确定的结论**(尤其涉及"对所有矩阵成立"的断言),宁可不写或标注"直观理解,严格证明见…",也不要拍脑袋下结论。

## 并行分组建议(避免依赖打架)

依赖关系:各篇的第 1 章主要依赖"第 1 篇(向量/组合/张成/无关/基)";篇内章节有顺序。

**推荐分批:**

1. **先写 P1-04(基与维数)** —— 收束第 1 篇,让地基(向量→组合→张成→无关→基/维数)完整。**这一章建议单独先写**,别并行。
2. **地基完整后,可并行开多个 agent**,每篇按内部顺序推进。互不冲突的最省心分法:**一个 agent 负责一篇**,篇内按章序写(P2→P2→…;P3→P3→…;…)。
   - 可同时启动的"各篇第一章":P2-05、P3-09、P4-12、P5-15、P6-18(都只依赖第 1 篇地基)。
3. **跨篇可并行**(不同篇之间相对独立),但**篇内按顺序**(如秩 P3-10 依赖行列式 P3-09;对角化 P4-13 依赖特征值 P4-12)。

> 每个 agent 拿到任务后,**先读总纲 + 自己篇相邻章 + 一篇样本**,再动笔。
