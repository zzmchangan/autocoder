# _章节写作提示词(给写作 agent)

本文件是写每一章前的标准纪律。每个写作 agent 动笔前必须读完本文件并照做。

---

## 一、写每章前必填的五要素

```
本章主线落点:这章在"统一与精简换小而快"里负责讲哪一招?
二分法位置:编译侧 / 执行侧 / 数据根基?
对照栏对象:CPython 的哪个点(指令模型 / GC / 数据模型)?
源码版本与锚点:lua-5.5.0,本章涉及文件与起始函数(带 file:line)
基调:纯直球,不用比喻
```

## 二、固定四段结构(每章按此组织)

1. **这章解决什么问题**:开篇直接抛出本章要回答的问题(为什么需要这个机制?不这样会怎样?),2–3 段,不绕弯,不用比喻。
2. **源码怎么实现**:贴关键结构体/函数(带 `file:line`),走一遍数据流/控制流。这是主体,占全章 50% 以上。
3. **为什么这样设计是 sound 的**:讲清楚这个设计为什么不丢正确性(三色不变式不漏不重 / 寄存器分配为什么不冲突 / open–closed upvalue 为什么不悬空)。这是"动机"命脉,对标 Tokio 的 why 双线。
4. **★对照 CPython + 回扣主线**:固定一栏,对照 CPython 的对应实现,点出 Lua 取舍的相对位置;结尾一句话回扣"统一与精简换小而快"。

## 三、纯直球纪律(本书招牌)

- **不用比喻**。不写"想象……""好比……""就像……"。直接讲数据结构、算法、不变式。
- 讲设计靠第一性原理:这个机制解决什么问题?约束是什么?为什么是这个解?
- 讲实现靠真实源码:函数名、行号、字段、宏,贴出来走一遍。

## 四、源码引用规范

- 版本:**lua-5.5.0**(唯一主干)。
- 引用格式:`lvm.c:1972`(`luaV_execute`)。关键函数给出 file:line。
- 代码块用三反引号标注 `c`。贴的代码必须与 5.5.0 源码逐字一致(可用 `...` 表示跳过无关行,但保留的每一行必须真实)。
- **交付前用 Grep/Read 核对每一个引用的行号**。5.5 与老资料(5.3/5.4)有差异,凡冲突处以源码为准并显式标注,例如:"5.5 中 Table 哈希实现已重写,与讲 5.3/5.4 的资料不同"。

## 五、标点规范

- 中文正文全角标点:。、:;""''。
- 代码、文件名、行号、变量名用半角。
- 全书零标点违规(主控抽查)。

## 六、对照栏规范

- 每章必须有 **★对照 CPython** 一栏(可独立小节,也可融入第 4 段)。
- 对照三刀任选贴切者:指令模型(寄存器 vs 栈)/ GC(增量标记 vs 引用计数+分代)/ 数据模型(统一 Table vs 多内建类型)。
- 对照要具体(给出 CPython 的对应机制与做法),不要泛泛说"Python 更复杂"。

## 七、行号核实清单(交付前自检)

- [ ] 本章引用的每个 `file:line` 已 Grep/Read 核实存在于 lua-5.5.0
- [ ] 贴的代码块与源码逐字一致
- [ ] 标点全角(代码除外)
- [ ] 有 ★对照 CPython 栏
- [ ] 结尾回扣主线
- [ ] 5.5 vs 老资料差异已标注
