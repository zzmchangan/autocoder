# 《Lua虚拟机设计与实现深入浅出》·目录与导读

> 主线:**统一与精简换小而快**。二分法:**编译侧 ↔ 执行侧**。★对照:**CPython**。源码:**lua-5.5.0**。基调:**纯直球,不用比喻**。

---

## P0 开篇

- **P0-01 第一性原理:为什么 Lua 又小又快又能塞进任何宿主**
  提出全书主线。讲清 Lua 的核心张力:必须小到能嵌入任何宿主,又必须快到不像脚本。Lua 的答案是三招——一切皆 Table、寄存器式字节码、增量 GC——用更少的机制换更多的能力。

## P1 值的统一(一切皆值)

- **P1-02 lua_State 与 TValue:万物皆值与 tagged 编码**
  一个 Lua 值在 C 里长什么样。`TValue`(`lobject.h:67`)用一个联合存值、用一个 tag 存类型,基本类型共用一种表示。`lua_State`(`lstate.h:285`)是一个 VM 实例的一切。
- **P1-03 字符串:短串驻留与长串惰性哈希**
  短字符串全局驻留(interning),指针比较即相等;长字符串惰性算哈希。统一接口下的两种策略。
- **P1-04 Table(上):数组与哈希的合体**
  `Table`(`lobject.h:776`)是 Lua 唯一的复合结构,数组部分与哈希部分同居。为什么这一个结构能充当数组、哈希、对象、命名空间。
- **P1-05 Table(下):哈希探测与 rehash**
  开放寻址(5.5 新实现)、装填因子、rehash 时数组与哈希两部分如何重新切分。

## P2 编译器:源码→字节码

- **P2-06 词法分析 llex:源码到 token**
  字符流怎么变成 token 流。保留字、数字、字符串、长括号。
- **P2-07 语法分析(上):递归下降与算符优先**
  `lparser` 不建 AST,边解析边生成代码。递归下降 + 算符优先表。
- **P2-08 语法分析(下):作用域与 upvalue 的静态收集**
  局部变量的栈槽分配、词法作用域、upvalue 在编译期就确定。
- **P2-09 代码生成 lcode:寄存器分配与指令发射**
  寄存器怎么分(线性分配)、指令怎么发射、常量表与跳转的回填。

## P3 字节码与寄存器式 VM

- **P3-10 指令格式 lopcodes:32 位编码与操作码全集**
  一条指令 32 位,ABC/ABx/AsBx/iABx/iAsBx/iAx 多种编码,操作码位宽与全部操作码一览。
- **P3-11 解释器循环 luaV_execute:取指-译码-执行-分发**
  VM 主循环。switch dispatch / goto jumptab 两种分发。一条指令的生命周期。
- **P3-12 寄存器式精解:为什么比栈式指令少(★对照 CPython)**
  用同一段 Lua 代码,对照 Lua 寄存器式字节码与 CPython 栈式字节码,数指令条数。寄存器式的代价与收益。

## P4 函数、闭包与调用

- **P4-13 调用栈与调用约定 luaD_call**
  函数怎么被调起来。CallInfo 帧、参数入栈、多返回值。
- **P4-14 闭包与 upvalue:词法捕获的开合**
  upvalue 是对外层局部变量的引用。open(指向栈)与 closed(拷出)两种状态。
- **P4-15 尾调用、可变参数与 to-be-closed**
  尾调用不增长栈(TAILCALL)、vararg 的特殊槽、to-be-closed 的析构钩子。

## P5 垃圾回收

- **P5-16 三色标记与增量步进:GC 必须可中断**
  三色不变式、增量步进为什么对嵌入至关重要(不能长时间卡宿主)。`lgc` 主循环。
- **P5-17 对象回收与弱表**
  字符串/表/闭包/upvalue/线程各自的标记,弱表(weak key/value)的语义。
- **P5-18 分代 GC**
  5.x 的分代模式:年轻代/老年代,为什么多数对象朝生夕死。

## P6 元表、协程与 C API

- **P6-19 元表与元方法:Table 如何充当对象**
  元表是 Table 的"类型行为"。元方法(`__index`/`__newindex`/`__add`…)让 Table 支持运算符重载和继承。
- **P6-20 协程:协作式多任务的栈切换**
  `coroutine.create`/`yield`/`resume`。协程靠切 `lua_State` 的栈实现,不是线程。
- **P6-21 C API 与嵌入:宿主与脚本的双向桥**
  `lua_State` 的栈式 C API(`lapi.c`)。宿主怎么压值、调函数、取结果;怎么把 C 函数注册给 Lua 调。嵌入的完整闭环。
