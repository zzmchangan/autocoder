# 《SQLite 设计与实现深入浅出:为什么把 SQL 编译成字节码再执行》—— 目录与导读

> 一本写给"天天用 SQLite(或在 App/浏览器里不知不觉用它)、却没想过它内部怎么跑"的人的小书。
>
> **一句话主旨**:SQLite 把 SQL 编译成 VDBE 字节码(opcode)、用虚拟机执行,执行时从单文件 B-tree 读数据、经 pager + WAL/journal 保证 ACID——"编译成字节码 + 虚拟机执行 + 单文件 B-tree"是它的本质。
>
> **二分法**(迷路时回到它):**编译与执行**(SQL 怎么变成字节码并执行:Tokenizer/Parser/Code Generator/VDBE) vs **存储与事务**(数据怎么存、怎么不丢:B-tree/Pager/WAL/VFS)。
>
> **主比喻**:直球为主、比喻点睛。开篇用"自带发动机的小车"做一次性点睛(MySQL/PG 是车队+加油站 C/S,SQLite 是自带发动机的小车,钥匙给你、开哪是哪),全书不再建立贯穿重比喻。
>
> **四重承接**:VDBE 字节码虚拟机↔《Lua》VM、B-tree/WAL↔《MySQL·InnoDB》、嵌入式↔《LevelDB》、C/S 对照↔《PG/MySQL》。

每章一行:**一句话钩子** —— 技巧标签 —— 二分法归属(`编译与执行` / `存储与事务` / `总览`)。
