## 一、这本书的定位

### 目标读者

会写程序、懂数学(微积分、线性代数)、懂系统(Linux 内核 / 数据库 / 分布式都写过或读过),**但物理引擎 / 游戏是零基础**。听说过"Box2D""PhysX""刚体""碰撞检测""约束"这些词,但讲不清:

- 游戏里物体掉下来、撞墙反弹、堆叠不穿透——这些"物理"是怎么实现的?是写了一堆 `if` 判断吗?当然不是。
- 物理引擎里的"物理"是真的物理吗?——**不是**。它是用**数值方法**(数值积分 + 几何检测 + 约束求解)做的**近似**,让物体每帧"看起来"符合物理。
- 一个物体的运动(位置、速度),物理引擎怎么每帧推进?为什么不能直接解析地算,非要用**数值积分**(欧拉、Verlet)?数值积分为什么会"发散爆炸",怎么选稳定的积分器?
- 两个物体有没有撞上,怎么判断?为什么分**宽相(粗筛谁可能碰)**和**窄相(精确算谁真碰)**两步?**SAT(分离轴)**、**GJK** 这些算法凭什么判断两个形状相交?
- 撞上之后呢?怎么让它们**不穿透**、按正确的方向**弹开**?**约束求解器(Sequential Impulse)** 在反复迭代什么?
- 物理引擎为什么用**固定步长**(游戏引擎那本讲过),变步长会怎样?

读者不是不聪明,是**没接触过这个领域**,缺的是地基。所以本书**不假设读者懂任何物理引擎术语**,从头打地基,但**假设读者懂微积分和线性代数**——这样讲到数值积分的稳定性、SAT 的投影、GJK 的闵可夫斯基差时,读者接得住。

### 全书一句话主旨

> **物理引擎,就是用数值方法让虚拟物体每帧"看起来"遵守物理:用数值积分推进运动(位置/速度),用几何算法检测碰撞(谁和谁碰了),用约束求解器响应碰撞(不穿透、正确弹开)。它的核心是"检测 vs 响应"二分——先检测(broad 宽相粗筛 → narrow 窄相精确),再响应(动力学积分 + 约束求解)。本质是数值近似的"假"物理,不是真物理。**

每个组件,都是这个"每步推进 + 检测 + 响应"流程的一环。读完一本书,读者能在脑子里放映出物理引擎一个时间步的发生过程:施加力 → 积分更新速度位置 → 宽相检测(AABB 粗筛)→ 窄相检测(SAT/GJK 精确 + 接触流形)→ 约束求解(Sequential Impulse 反复迭代让不穿透)→ 进入下一步——以及每一步**为什么必须存在、原理是什么、源码怎么实现**。

### 为什么用"检测 vs 响应"当骨架(而非按 Box2D 模块平铺)

很多物理引擎书的写法是"按概念讲":刚体、力、碰撞、关节……本书**不这么写**。因为那样讲,读者记的是"有哪些概念",而不是"物理引擎每步在解决什么问题"。本书用物理引擎**真正的两半**当骨架——**检测(判断谁碰了)vs 响应(碰了怎么动)**。检测又分宽相粗筛 → 窄相精确,响应又分动力学积分 → 约束求解。每一步都是这个流程上解决一个必须解决的问题。读者永远知道**我在哪、为什么需要下一步**。

## 三、全书主线、二分法、主比喻、承接

### 主线

> **物理引擎 = 用数值方法让虚拟物体每帧"看起来"符合物理。核心流程:积分推进运动 → 检测碰撞(broad 粗筛 → narrow 精确)→ 约束求解响应。本质是数值近似的"假"物理。**

任何一处看不懂某个机制,回到这句问:"这是在'检测'(判断谁碰了:宽相 / 窄相 / 接触流形),还是在'响应'(碰了怎么动:积分 / 约束求解)?它在解决'运动''相交判断''不穿透''正确弹开'中的哪个问题?"

### 二分法

本书的二分法,对应物理引擎两大主题:

> **检测(碰撞检测:谁和谁碰了) vs 响应(动力学 + 约束:碰了之后怎么动)。**

- **检测这一面**:进一步分两步——**宽相(broad phase)**:用 AABB 包围盒 + 空间划分(动态 AABB 树 / 扫掠剪枝 / 网格)快速排除绝大多数不可能碰的对,只留下少数"可能碰"的候选;**窄相(narrow phase)**:对候选对用 **SAT**(分离轴)/ **GJK**(闵可夫斯基差)精确判断是否真相交,并算出**接触流形**(接触点、法线、穿透深度)。检测决定"谁碰了、碰在哪、多深"。
- **响应这一面**:进一步分两层——**动力学**:用数值积分(半隐式欧拉 / Verlet)施加力、更新速度和位置;**约束求解**:碰撞约束(不穿透)+ 关节约束(铰链、距离),用 **Sequential Impulse**(顺序冲量法 / PGS)反复迭代求解。响应决定"碰了之后物体怎么动,且不违反物理约束"。
- **横切两面**:固定时间步长(承游戏引擎,保证数值稳定)、休眠(静止物体不计算)、连续碰撞 CCD(防高速穿透)。

任何时候迷路,回到这个二分法:**"这是在检测碰撞,还是在响应碰撞?"**

### 主比喻决策:数值钟表 + 几何裁判,直球为主、比喻点睛

> **直球为主**:绝大多数地方第一性原理 + 数学推导直给。**比喻点睛**:在开篇(P0-01)用"**一帧一帧拨的数值钟表**"+"**几何裁判**"做一次性点睛——物理引擎不像真实世界那样物理"连续"地发生,而是像一块每帧"咔哒"拨一格的钟表,每咔哒一声(一个时间步),数值积分推进一下运动、几何裁判(检测)判一下谁碰了、约束求解让它们规矩地动;真实物理是连续的,引擎是离散近似的。全书不建立贯穿重比喻。

### 承接(本书最大特色,务必遵守)

本书是子线第三本,承接网络密集:

- **★承《数学分析》**(最强承接):物理引擎的灵魂是**数值方法**——数值积分(半隐式欧拉 / Verlet / RK4)是《数学分析》"精确 vs 逼近"在离散时间上的极致应用;约束求解(Sequential Impulse / PGS)本质是解线性互补问题(LCP),是数值线性代数。本书第 2 篇(P2-06~08)把这条承接兑现——读者会发现"物理引擎的稳定性问题,就是《数学分析》讲的数值方法稳定性"。
- **★承《线性代数》**:SAT 用**投影**(向量点乘)判断分离轴、GJK 用**闵可夫斯基差 + 单纯形**、刚体的惯性张量是矩阵。本书第 4 篇(P4-12~13)呼应线代的向量 / 投影。
- **承《游戏引擎》**:物理是游戏引擎主循环 update 里**最重的一段**,且用**固定步长**(游戏引擎那本 P3-10 讲过)。本书 P2-08 一句带过指路 [[graphics-series-project]],篇幅留物理特有的稳定性。
- **承《Linux 调度器》《Tokio》**:宽相的空间划分(动态 AABB 树)是数据结构,可对照分配器的树/调度器的堆(弱承接,一句带过)。

### 反哺 / 横连

- 本书讲透数值积分后,**反哺**读者对《数学分析》"数值逼近"的理解——同一思想(离散近似连续)的不同场景。
- 物理引擎的"检测 vs 响应"模型,对照**碰撞回避(机器人/自动驾驶)**、**布料/流体仿真**、**CAD 装配干涉检查**——本质都是"几何检测 + 物理响应"。

## 四、全书结构(分篇 + 分章)

> 每章给**章名 + 核心问题 + 关键源码 / 数学模块 + 本章典型技巧 + 二分法归属**。全书旅程:**先打地基(物理引擎是什么 + 数值积分直觉,第 0~2 篇)→ 再检测(宽相 → 窄相,第 3~4 篇)→ 再响应(约束求解,第 5 篇)→ 收束(第 6 篇)**。
> 章号全局连续;命名 `P{篇}-{章}-标题.md`。

### 第 0 篇 · 开篇:物理引擎是什么(定调,1 章)

- **第 1 章 · 第一性原理:虚拟物体凭什么看起来遵守物理**(`P0-01`):从"游戏里物体掉落反弹堆叠"切入——这些"物理"不是写 if 判断,而是数值方法。物理引擎的"物理"是**数值近似的假物理**:数值积分推进运动、几何检测判断碰撞、约束求解响应。建立"检测 vs 响应"二分和每步流程。**【样本章节,全书风格标杆】**

### 第 1 篇 · 物理引擎鸟瞰:新手地基(3 章)

> 给零基础读者打地基。讲清物理引擎全貌和"为什么是数值方法"。

- **第 2 章 · 物理引擎在干什么:牛顿方程与离散时间**(`P1-02`)——真实物理是连续的(F=ma 是微分方程),可计算机每帧离散推进。物理引擎 = 把连续物理方程,变成每时间步离散更新。归属:`响应`。
- **第 3 章 · 为什么物理要数值积分**(`P1-03`)——为什么不解析地解运动方程(大多数情况解不出),非要用数值积分?承《数学分析》数值方法。归属:`响应`。
- **第 4 章 · 物理引擎的两半:检测 vs 响应**(`P1-04`)——一个时间步的完整流程鸟瞰:积分推进 → 宽相检测 → 窄相检测 → 约束求解。建立全书二分法骨架。归属:`总览`。

### 第 2 篇 · 运动与积分:响应侧·动力学基础(4 章,承数学分析招牌)

> 物理引擎的灵魂:怎么数值地推进运动。主载体 Box2D v3 的积分器。承《数学分析》。

- **第 5 章 · 刚体动力学:质量、惯性、力矩**(`P2-05`)——物体运动的物理量:位置 / 速度 / 加速度、质量、惯性张量(承《线性代数》矩阵)、力与力矩。归属:`响应`。
- **第 6 章 · 显式欧拉及其不稳定**(`P2-06`)——最朴素的积分器,凭什么**能量发散爆炸**(承《数学分析》数值稳定性)。配轨迹图。归属:`响应(招牌)`。
- **第 7 章 · 半隐式欧拉与 Verlet**(`P2-07`)——Box2D 用的**半隐式欧拉**(symplectic Euler)凭什么稳定、**Verlet** 积分怎么天然处理约束。技巧:辛积分器保能量。源码:Box2D v3 的积分。归属:`响应(招牌)`。
- **第 8 章 · 固定步长与稳定性**(`P2-08`)——为什么物理用**固定步长**(数值稳定、可复现),承《游戏引擎》P3-10 一句带过。变步长撞什么墙。归属:`响应`。

### 第 3 篇 · 碰撞检测·宽相:检测侧·粗筛(3 章)

> 检测的第一步:从海量物体对里,快速排除不可能碰的,留少数候选。承空间划分。

- **第 9 章 · AABB:轴对齐包围盒**(`P3-09`)——用一个轴对齐的矩形/长方体"包住"任意形状,快速判断两物体"可能碰"。技巧:AABB 相交的快速判断(逐轴投影)。归属:`检测`。
- **第 10 章 · 宽相:空间划分**(`P3-10`)——海量物体两两配对是 O(n²),怎么用空间划分(网格 / 树 / 扫掠剪枝)降到近 O(n)?承渲染管线 BVH、游戏引擎空间划分。归属:`检测(招牌)`。
- **第 11 章 · 动态 AABB 树**(`P3-11`)——Box2D 宽相的招牌:一棵随物体运动增量更新的平衡树。技巧:树的重平衡、叶子旋转。源码:Box2D v3 的 broad phase 树。归属:`检测(招牌)`。

### 第 4 篇 · 碰撞检测·窄相:检测侧·精确(3 章,承线性代数)

> 检测的第二步:对候选对精确判断是否真相交,并算接触细节。承《线性代数》。

- **第 12 章 · SAT:分离轴定理**(`P4-12`)——两个凸形状,凭什么"存在一条轴把它们投影分开就不相交"?技巧:★SAT 的投影判断(承《线性代数》向量投影)。归属:`检测(招牌)`。
- **第 13 章 · GJK 与闵可夫斯基差**(`P4-13`)——两个凸形状的距离 / 相交判断:闵可夫斯基差 + 单纯形迭代。技巧:★GJK 算法(承《线性代数》凸集)。归属:`检测(招牌)`。
- **第 14 章 · 接触流形:法线、穿透、接触点**(`P4-14`)——相交后算出碰撞的几何细节:接触法线(沿哪个方向弹)、穿透深度(多深)、接触点(在哪)。为约束求解做准备。归属:`检测`。

### 第 5 篇 · 碰撞响应·约束求解:响应侧·核心(4 章,全书最难)

> 响应的核心:碰了之后怎么不穿透、正确动。物理引擎最难的部分,主载体 Box2D 的 Sequential Impulse。

- **第 15 章 · 冲量法碰撞响应**(`P5-15`)——碰撞后速度怎么变?技巧:**冲量(impulse)** 瞬间改变速度、恢复系数(弹性)。配碰前碰后速度图。归属:`响应(招牌)`。
- **第 16 章 · Sequential Impulse 约束求解**(`P5-16`)——多个约束(多接触点、堆叠)同时要满足,怎么解?技巧:★**顺序冲量法 / PGS**(Box2D 招牌,Erin Catto),反复迭代修正违反的约束,收敛到满足所有约束。本质是解 LCP。源码:Box2D v3 的约束求解。归属:`响应(招牌)`。
- **第 17 章 · 关节约束**(`P5-17`)——铰链、距离、滑轮这些关节怎么约束两个物体?技巧:关节 = 一种约束,同样进 Sequential Impulse 求解。归属:`响应`。

## 六、写作约定(沿用《图形渲染管线》《游戏引擎》系列)

1. **新手向"先基础后源码"**(本书命脉):每个概念先讲"是什么 / 为什么 / 数学直觉",再上源码。基础章节宁厚勿薄。
2. **动机 + 技巧双线**(本书命脉):每个关键点 why 与 how 并重,每章单开"技巧精解",配反面对比("朴素做法 / 不稳定积分撞什么墙")。
3. **"为什么"永远先于"怎么做"**:先讲它解决什么本质问题、不这样会怎样(如显式欧拉怎么爆炸),再讲实现。
4. **承接数学线**:数学分析(数值积分)、线性代数(投影 / 凸集)讲透的呼应指路,篇幅留物理特有的。
5. **大白话 + 数学 / 几何直觉 + 比喻点睛**:第一性原理直给,配几何 / 轨迹图;"数值钟表 + 几何裁判"比喻只在 P0-01 点睛。
6. **每章标准结构**:① 章首(核心问题 + 读完你会明白 + 难度高的加逃生阀);② 一句话点破;③ 正文若干小节(招牌四段式:**提问 → 朴素做法 / 不稳定撞什么墙 → 所以这么设计 → 源码 / 数学佐证**);④ 技巧精解;⑤ 章末小结(回扣"检测 vs 响应"主线 + 五个为什么清单 + 想深入往哪钻 + 引出下一章)。
7. **回扣全局**:每章结尾回到二分法,说明本章服务"检测"还是"响应",衔接前后章。
8. **语言**:中文,技术名词保留英文(刚体 rigid body、AABB、broad/narrow phase、SAT、GJK、Minkowski difference、impulse、Sequential Impulse / PGS、constraint、CCD、Verlet / symplectic Euler 等)。
9. **标点**:**半角 ASCII 逗号 `,`、冒号 `:`、括号 `()`、问号 `?`;全角顿号 `、`、破折号 `——`、引号 `""`、句号 `。`**;中英文之间留空格。
10. **文件命名**:`P{篇}-{章}-标题.md`。
11. **★配图用 matplotlib PNG**(不用 ASCII):数值积分轨迹、AABB / SAT 投影 / GJK 单纯形 / 接触流形 / 冲量,用 PNG;流程图用 mermaid。每章至少 2~3 张。
