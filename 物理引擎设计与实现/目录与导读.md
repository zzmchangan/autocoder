# 《物理引擎设计与实现深入浅出:虚拟物体凭什么看起来遵守物理》—— 目录与导读

> 一本写给"会写代码、懂数学(微积分、线性代数)、甚至读过内核 / 数据库源码,**但完全没接触过物理引擎和游戏**"的工程师的小书。
>
> **一句话主旨**:物理引擎 = 用数值方法让虚拟物体每帧"看起来"符合物理;核心流程:积分推进运动 → 检测碰撞(broad 粗筛 → narrow 精确)→ 约束求解响应。本质是数值近似的"假"物理。
>
> **二分法**(迷路时回到它):**检测**(碰撞检测:谁碰了,broad 宽相粗筛 → narrow 窄相精确) vs **响应**(动力学积分 + 约束求解:碰了怎么动)。
>
> **承接**:★承《数学分析》(数值积分 / 约束求解 LCP,物理灵魂)、★承《线性代数》(SAT 投影 / GJK 闵可夫斯基差 / 惯性张量)、承《游戏引擎》(物理是 update 最重的一段、固定步长)。
>
> **主比喻**:直球为主、比喻点睛。开篇用"一帧一帧拨的数值钟表"+"几何裁判"做一次性点位睛。

每章一行:**一句话钩子** —— 技巧标签 —— 二分法归属(`检测` / `响应` / `横切` / `总览`)。

> **新手向提醒**:第 0~2 篇(P0-01 ~ P2-08,共 8 章)是**原理地基**,全力打基础、暂不深入源码,**强烈建议顺序读、不要跳**;第 3 篇起进入源码精解。零基础读者从 P0-01 顺着读最省力。
