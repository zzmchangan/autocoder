# 《游戏引擎设计与实现深入浅出:一个虚拟世界怎么每帧更新 60 次》—— 目录与导读

> 一本写给"会写代码、懂数据结构与算法、甚至读过内核 / 数据库源码,**但完全没接触过游戏引擎和客户端**"的工程师的小书。
>
> **一句话主旨**:游戏引擎 = 每秒把虚拟世界状态推进 60 次并渲染的大循环;核心架构难题 = 组织海量对象让每帧够快;答案 = ECS + 数据导向设计(按"系统怎么遍历"布局数据)。
>
> **二分法**(迷路时回到它):**组织**(ECS:数据怎么躺,面向缓存 / 并行) vs **驱动**(主循环 / 时间 / 并发:循环怎么跑,面向实时 / 帧率)。
>
> **承接**:★强承《图形渲染管线》(渲染子系统 = 那条管线,一句带过)、★承《内存分配器》(ECS 的 SoA / 缓存行 / SIMD / 数据导向,最精彩)、承《Lua》(脚本热重载)、承《JVM》(Unity C# / IL2CPP)。
>
> **主比喻**:直球为主、比喻点睛。开篇用"永不停止的 while 循环"+"按取用习惯摆放的仓库"做一次性点位睛。

每章一行:**一句话钩子** —— 技巧标签 —— 二分法归属(`组织` / `驱动` / `横切` / `总览`)。

> **新手向提醒**:第 0~2 篇(P0-01 ~ P2-09,共 9 章)是**原理地基**,全力打基础、暂不深入源码,**强烈建议顺序读、不要跳**;第 3 篇起进入源码精解。零基础读者从 P0-01 顺着读最省力。
